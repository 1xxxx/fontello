Comments on archive content
---------------------------

- <font_name>.(woff|eot|ttf|svg) - fonts in different formats

- <font_name>.css - css, with fontname definitions. Should be ok with 
  twitter bootstrap

- demo.html - demo file, to show your webfont content

- <font_name>-codes.css - css with character codes only, if you like to make
  custom @font-face definition. For example, if you use assets matager to build
  static css

- config.json - keeps your settings. You can import it back to fontello anytime,
  to continue your work
